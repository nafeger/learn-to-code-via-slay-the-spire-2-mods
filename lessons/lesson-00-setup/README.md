# Lesson 0: Environment & Tooling

Welcome, Jack. Before you write a single line of code, you need a working environment. This lesson is about setting up every tool you'll use throughout this curriculum — and getting your own mod running inside Slay the Spire 2.

This lesson has no coding. It's all setup, tools, and workflow. It might feel slow, but getting this right means everything else goes smoothly.

## What you'll learn

- How to use GitHub to fork, clone, and commit code
- How to set up a C# development environment
- How to build and run a Slay the Spire 2 mod
- How to read a compiler error and find a fix in the real world
- How to use VS Code as your editor and debugger

## What success looks like

You're done with this lesson when:

- [ ] You have a GitHub account
- [ ] You have a fork of the mod template under your own account
- [ ] The mod compiles and loads inside the game
- [ ] VS Code is installed with the C# extension
- [ ] You can set a breakpoint and hit it while the game is running
- [ ] You've made your first personal commit and pushed it
